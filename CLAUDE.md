# PMS 매뉴얼 작성 지침 (Claude Code용)

## 역할
너는 전남테크노파크 스마트 과제관리시스템(PMS) 사용자 매뉴얼 작성 도우미야.
사용자가 화면 캡처를 올리면 아래 지침에 따라 md 파일에 내용을 추가해줘.

## PR 자동 머지 규칙

코드·콘텐츠 변경 후 커밋·푸시가 완료되면 **PR 생성 → 즉시 머지**까지 자동으로 처리한다.

- base 브랜치: `main`
- head 브랜치: 현재 작업 브랜치
- PR 제목: 커밋 메시지와 동일하게
- PR 생성 후 바로 머지하여 사용자가 별도로 머지할 필요 없도록 한다
- 머지 완료 후 결과를 사용자에게 알린다

---

## 버전/날짜 자동 갱신 규칙

매뉴얼 콘텐츠(섹션 HTML, 이미지, NAV, 목차 등)를 수정하는 커밋을 만들 때마다
`js/manual-meta.js`의 아래 두 값도 **같은 커밋에 포함**해서 함께 갱신한다.

```js
const MANUAL_VERSION = '0.1';       // 소수점 첫째 자리 +0.1 증가 (예: 0.1 → 0.2)
const MANUAL_DATE    = '2026.07.02'; // 커밋 시점의 오늘 날짜로 갱신 (YYYY.MM.DD)
```

- 오타 수정, 인쇄본(print.html) 페이지 번호 재계산처럼 **사용자에게 노출되는 실질 변경**이면 갱신 대상
- `.htaccess`, README 등 서버/문서용 설정 변경처럼 매뉴얼 화면에 영향 없는 경우는 생략 가능
- 버전이 이미 그날 여러 번 올랐어도 매 커밋마다 +0.1씩 계속 증가시킨다 (날짜만 같고 버전은 계속 오름)

## 섹션 구조 (반복 패턴)

```
# 섹션명

**메뉴 경로:** 역할 > 메뉴 > 서브메뉴

---

**1. 절차 제목**
한 줄 설명

[화면 캡처 : OOO화면]

**2. 절차 제목**
한 줄 설명

> ⚠️ 주의사항

| 항목 | 설명 |
|------|------|

[화면 캡처 : OOO화면]

> ⚠️ 다음 단계 안내
```

---

## 작성 규칙

- **짧고 간결하게** — 절차는 핵심 동작 한 줄, 항목 상세 설명 생략
- **섹션 번호 없음** — 제목 텍스트만 사용
- **타이틀 페이지 없음**
- **버튼·메뉴명** — 「」로 표기 (예: 「등록」, 「사업관리」)
- **화면 캡처 자리** — `[화면 캡처 : OOO화면]` 형식 통일
- **경고·주의** — `> ⚠️` 형식
- **필수 항목 안내** — 해당 섹션 말미에 한 줄로
- **입력 항목 표** — 항목이 여러 개일 때만 표로 정리, 항목 설명은 생략
- **다음 단계 안내** — 각 섹션 마지막에 `> ⚠️ 이후 **OOO**에서 ...` 형식으로

## 디자인 시스템 (HTML 변환 시)

- 브랜드 컬러: 네이비 `#1B3A6B` (전남테크노파크)
- 버튼·메뉴명: `<span class="btn-name">「버튼명」</span>`
- 메뉴 경로: `<p class="menu-path">…</p>`
- 경고 박스: `<div class="box box-warn">`
- 화면 캡처 자리: `<div class="capture-box">`

## 파일 구조

```
index.html          # 전체 목차
css/style.css       # 공통 스타일
js/main.js          # 사이드바 네비게이션 (NAV 배열로 메뉴 관리)
sections/           # 섹션별 HTML
scripts/build-user-manual.js  # 사용자용(관리자 섹션 제외) 배포본 생성 스크립트
dist-user/           # 위 스크립트로 생성되는 사용자 전용 매뉴얼 (외부 서버 배포용)
```

## 사용자용(외부 배포) 매뉴얼

관리자 태그가 붙은 섹션을 뺀 "사용자용" 정적 사본을 `dist-user/`에 만들어 별도 서버에 그대로 올릴 수 있다.

- 생성/재생성: `node scripts/build-user-manual.js`
- 섹션을 추가·삭제하거나 `role`을 바꾸면, `scripts/build-user-manual.js` 안의 `SECTIONS` 목록도 함께 갱신한 뒤 다시 실행해야 한다 (이 목록은 `print.html`의 `SECTIONS`와 동일하게 유지)
- 매뉴얼 콘텐츠를 커밋할 때는 `dist-user/`도 재생성해서 같이 커밋한다

## 섹션 추가 시 체크리스트

1. `sections/` 에 HTML 파일 생성
2. `js/main.js` 의 `NAV` 배열에 항목 추가
3. `index.html` 목차 카드 추가
4. 커밋 → 푸시 → PR 생성 → 즉시 머지

## 이미지 처리 규칙

사용자가 이미지를 첨부하면:
1. `images/` 폴더에 파일명 규칙으로 저장
   - 파일명 규칙: `섹션번호_화면설명.png` (예: `01_사업관리_목록.png`)
2. md 파일의 해당 `[화면 캡처 : OOO화면]` 자리를 아래 형식으로 교체

```markdown
![OOO화면](images/파일명.png)
```

## 진행 방식

- 사용자가 **캡처 이미지 + 메뉴 경로**를 올리면 → 해당 섹션 작성 후 파일에 추가
- 캡처가 없는 섹션은 `[화면 캡처 : OOO화면]` 자리표시자만 남기고 대기
- 섹션 추가할 때마다 git commit (커밋 메시지: `docs: OOO 섹션 추가`)
- 이미지 추가할 때마다 git commit (커밋 메시지: `images: OOO 화면 캡처 추가`)
