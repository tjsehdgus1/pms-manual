# 전남테크노파크 스마트 과제관리시스템 (PMS) 역기획 PRD

> 작성일: 2026-05-15  
> 작성 목적: 70% 개발 완료된 시스템을 역기획하여 사용자 매뉴얼 작성의 기준 자료로 활용  
> 기술 스택: Java / Spring MVC / eGovFramework / JSP / MyBatis / Tomcat 9.0

---

## 1. 시스템 개요

### 1.1 프로젝트 목적
전남테크노파크가 지원하는 기업들의 **과제(사업) 신청·관리·평가** 전 과정을 온라인으로 처리하는 통합 과제관리시스템.

### 1.2 주요 이해관계자 (사용자 역할)

| 역할 | 설명 | 접근 영역 |
|------|------|----------|
| **관리자** | 전남테크노파크 운영 담당자 | 백오피스 전체 (`/_msys`) |
| **기업 사용자** | 과제를 신청하는 기업 담당자 | 마이페이지 (`/_m/myp`) |
| **심사자 / 평가위원** | 신청 과제를 심사하는 외부 전문가 | 마이페이지 심사뷰 |
| **일반 방문자** | 공고 및 공지 열람 | 공개 포털 (`/_m/ko`) |

### 1.3 시스템 구성 (3개 영역)

```
jntppms
├── /_m/ko          → 공개 포털 (사업공고, 공지사항, FAQ 등)
├── /_m/myp         → 기업 마이페이지 (과제신청, 서류 제출, 심사)
└── /_msys          → 관리자 백오피스 (사업관리, 공고관리, 평가관리)
```

## 2. 기능 요구사항

### 2.1 공개 포털 (`/_m/ko`)

#### 2.1.1 메인 페이지
- 팝업 공지 표시
- 배너 광고 표시
- 최신 공지사항 목록 (AJAX)
- 지원사업 공고 목록 (AJAX)
- 타기관 공고 목록 (AJAX)

#### 2.1.2 회원가입 (7단계)
| 단계 | 화면 | 내용 |
|------|------|------|
| Step 1 | `join_step01.jsp` | 이용약관 동의 |
| Step 2 | `join_step02.jsp` | 개인정보처리방침 동의 |
| Step 3 | `join_step03.jsp` | 본인인증 (공동인증서) |
| Step 4 | `join_step04.jsp` | 기업 선택 / 신규 기업 등록 |
| Step 5 | `join_step05.jsp` | 기업정보 입력 |
| Step 6 | `join_step06.jsp` | 담당자 정보 입력 |
| Step 7 | `join_step07.jsp` | 가입 완료 |

#### 2.1.3 로그인
- 아이디 / 비밀번호 입력
- 아이디 찾기 / 비밀번호 재설정
- 로그인 후 마이페이지 이동

#### 2.1.4 게시판
| 메뉴 | 기능 |
|------|------|
| 공지사항 | 목록 조회, 상세 보기 |
| FAQ | 목록 조회, 상세 보기 (카테고리별) |
| 자료실 | 파일 다운로드 |
| 사업일정 | 월별 일정, 연간 일정 |
| 지원사업 공고 | 진행중/종료 공고 목록, 공고 상세 |
| 설문조사 | 목록, 참여 |

### 2.2 기업 마이페이지 (`/_m/myp`)

#### 2.2.1 대시보드
- 진행 중인 과제신청 현황 (최근 10건)
- 상태별 현황 요약

#### 2.2.2 과제신청 (`/myp/suptpro`) — 핵심 기능

**과제 신청은 6단계(Step)로 구성**

| Step | 화면 | 주요 내용 | 수정 가능 시점 |
|------|------|----------|--------------|
| 1 | `step1` (프로그램 정보) | 사업공고명, 프로그램명, 지원분야, 기술분류, 총사업비 | 읽기 전용 |
| 2 | `step2` (기업정보) | 사업자번호, 회사명, 설립일, 임직원 현황, 재무정보(3년) | 신청완료 전 |
| 3 | `step3` (적격성진단) | 항목별 Y/N 체크리스트 | 신청완료 전 |
| 4 | `step4` (컨소시엄/참여기관) | 주관사/참여사 정보, 참여기관 추가·삭제·상태변경 | 참여사 완료 전 |
| 5 | `step5` (사업정보) | 과제명, 기간, 예산, 사업계획서 파일 업로드 | 신청완료 전 |
| 6 | `step6` (요건검토) | 보완요청 확인, 보완자료 업로드, 심사 코멘트 | 보완요청 후 |

**과제신청 상태값**

| 상태코드 | 명칭 | 설명 |
|---------|------|------|
| `S0` | 작성중 | 초기 작성 단계 |
| `S1` | 신청완료 | 기업이 신청 완료 |
| `S2` | 요건검토중 | 관리자가 검토 시작 |
| `S3` | 보완요청 | 심사자가 보완 요청 |
| `S4` | 최종승인 | 주관사 최종 승인 |

**참여기관(컨소시엄) 상태값**

| 상태코드 | 명칭 |
|---------|------|
| `S1` | 참여신청 |
| `S3` | 참여완료 |
| `S4` | 승인완료 |

#### 2.2.3 파일 업로드 규정

| 구분 | 파일 종류 | 최대 용량 |
|------|----------|---------|
| Step 2 기업정보 | 사업자등록증, 직원 확인서, 재무제표(3년) | 30MB |
| Step 5 사업정보 | 사업계획서, 지원양식 | 100MB |
| Step 6 보완자료 | 보완 요청 답변 파일 | 30MB |

#### 2.2.4 기업정보 관리 (`/myp/corp`)
- 기업 기본정보 조회/수정
- 담당자 목록 관리

#### 2.2.5 사용자 정보 관리 (`/myp/user`)
- 개인정보 수정
- 비밀번호 변경

#### 2.2.6 설문조사 (`/myp/dmsurvey`)
- 수신된 설문 목록 조회
- 설문 참여 및 제출

#### 2.2.7 타기관 공고 (`/myp/otherannce`)
- 공고 목록 조회
- 공고 상세 보기

## 3. 비기능 요구사항

### 3.1 보안
- 로그인 필수 (세션 기반 인증)
- 역할별 접근 제어 (authLv 기반)
- 공동인증서 본인인증 (회원가입 시)
- IP 차단 기능 (관리자 설정)

### 3.2 파일 관리
- 최대 업로드: 일반 30MB, 사업계획서 100MB
- 허용 확장자: PDF, HWP, DOCX, XLSX, JPG, PNG, ZIP
- 파일 그룹(dataGrp) + 서브그룹(dataSubGrp)으로 분류 저장

### 3.3 알림
- 이메일 자동 발송 (신청 완료, 보완 요청, 승인 등)
- SMS 자동 발송 (주요 상태 변경 시)

## 4. 데이터 모델 (핵심 테이블)

```
BIS (사업)
├── BIS_PROJ (프로젝트)
│   └── BIS_PROJ_ANNCE (공고)
│       ├── BIS_PROJ_ANNCE_PROG (프로그램)
│       │   ├── BIS_PROJ_ANNCE_PROG_TYPE (NRND 분류)
│       │   └── BIS_PROJ_ANNCE_PROG_RND (RND 예산 항목)
│       └── ANNCE_REG_FILE (공고 첨부파일)
│
PROG_ORDER (과제신청)
├── PROG_ORDER_PART_CORP (참여기관)
│   └── PROG_ORDER_CORP_INFO (기업별 정보)
├── PROG_ORDER_AUDIT (적격성진단)
├── PROG_ORDER_RND (RND 신청 상세)
└── PROG_ORDER_REQ_CK (요건검토/보완요청)

CORP (기업)
└── CORP_USER (기업 사용자)

EVAL_POOL (평가위원 풀)
```
