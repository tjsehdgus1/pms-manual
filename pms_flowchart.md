## 1. 시스템 전체 구성도

```
┌─────────────────────────────────────────────────────────────────┐
│                  전남테크노파크 PMS 시스템                         │
├─────────────────┬───────────────────┬───────────────────────────┤
│   공개 포털      │   기업 마이페이지   │      관리자 백오피스         │
│  (/_m/ko)       │   (/_m/myp)        │      (/_msys)              │
├─────────────────┼───────────────────┼───────────────────────────┤
│ • 메인 페이지    │ • 대시보드         │ • 대시보드                  │
│ • 사업공고 조회  │ • 과제신청 (6단계) │ • 사업/공고 관리            │
│ • 공지사항       │ • 기업정보 관리    │ • 과제신청 심사             │
│ • FAQ           │ • 사용자 정보 관리 │ • 평가 관리                 │
│ • 자료실         │ • 설문조사 참여    │ • 게시판 관리               │
│ • 사업일정       │ • 타기관 공고 조회 │ • 메시지 발송               │
│ • 회원가입/로그인│                   │ • 통계/로그                 │
└─────────────────┴───────────────────┴───────────────────────────┘
```

**접속 방법:**
- 공개 포털: 웹브라우저에서 시스템 URL 접속
- 마이페이지: 로그인 후 자동 이동
- 관리자: 별도 관리자 URL 접속 (운영자에게 문의)

## 2. 회원가입 순서도

```mermaid
flowchart TD
    A([시작: 사이트 접속]) --> B[회원가입 버튼 클릭]
    B --> C[Step 1: 이용약관 동의]
    C --> D{약관 동의?}
    D -- 미동의 --> E[동의 필요 안내]
    E --> C
    D -- 동의 --> F[Step 2: 개인정보처리방침 동의]
    F --> G{동의?}
    G -- 미동의 --> H[동의 필요 안내]
    H --> F
    G -- 동의 --> I[Step 3: 본인인증\n공동인증서 인증]
    I --> J{인증 성공?}
    J -- 실패 --> K[인증 오류 안내]
    K --> I
    J -- 성공 --> L[Step 4: 기업 선택]
    L --> M{기업 존재?}
    M -- 기존 기업 검색 --> N[기업명/사업자번호 검색]
    N --> O[기업 선택]
    M -- 신규 등록 --> P[신규 기업 정보 입력]
    O --> Q[Step 5: 기업정보 확인/수정]
    P --> Q
    Q --> R[Step 6: 담당자 정보 입력\n아이디, 비밀번호, 이메일, 연락처]
    R --> S{필수항목 입력 완료?}
    S -- 미완료 --> T[입력 안내]
    T --> R
    S -- 완료 --> U[Step 7: 가입 완료]
    U --> V([로그인 페이지로 이동])
```

## 3. 로그인 순서도

flowchart TD

    C --> D[로그인 버튼 클릭]
    D --> E{인증 확인}
    E -- 실패 --> F[오류 메시지 표시\n'아이디 또는 비밀번호가 틀립니다']

    E -- 성공 --> G{사용자 역할 확인}

    B --> K[아이디 찾기 클릭]
    K --> L[이름 + 이메일 입력]
    L --> M[아이디 확인]
    
    B --> N[비밀번호 재설정 클릭]
    N --> O[아이디 + 이메일 입력]
    O --> P[임시 비밀번호 이메일 발송]

## 4. 기업 과제신청 순서도 (전체 흐름)

```mermaid
flowchart TD
    A([로그인 완료]) --> B[마이페이지 대시보드]
    B --> C[지원사업 공고 확인]
    C --> D[공개 포털에서 공고 상세 조회]
    D --> E{신청 가능 공고?}
    E -- 마감 --> F[신청 불가 안내]
    E -- 진행중 --> G[과제신청 버튼 클릭]
    G --> H[마이페이지 → 과제신청 메뉴]
    H --> I[신청서 작성 시작 버튼]
    I --> J[프로그램 선택]
    J --> K[Step 1: 프로그램 정보 확인]
    K --> L[Step 2: 기업정보 입력/확인]
    L --> M[Step 3: 적격성진단 작성]
    M --> N{컨소시엄 신청?}
    N -- 단독 신청 --> O[Step 4 건너뜀]
    N -- 컨소시엄 --> P[Step 4: 참여기관 추가]
    P --> Q[참여기관 담당자가 Step 4 완료]
    Q --> R[Step 5: 사업정보 입력]
    O --> R
    R --> S[사업계획서 등 파일 업로드]
    S --> T[Step 6: 요건검토 확인]
    T --> U{모든 단계 완료?}
    U -- 미완료 --> V[해당 단계로 돌아가 수정]
    V --> U
    U -- 완료 --> W[신청완료 버튼 클릭]
    W --> X{최종 제출 확인}
    X -- 취소 --> Y[계속 수정]
    Y --> U
    X -- 확인 --> Z[신청완료 상태 S1 변경]
    Z --> AA[이메일/SMS 발송]
    AA --> AB([신청 완료])
```

## 7. 관리자 과제 심사 순서도

```mermaid
flowchart TD
    A[신청관리 메뉴] --> B[신청서 목록\n상태: S1 신청완료]
    B --> C[신청서 클릭]
    C --> D[심사뷰로 신청서 확인\nStep 1~6 전체]
    D --> E{검토 결과}
    E -- 적합 --> F[상태 변경: S2 검토중]
    E -- 보완 필요 --> G[보완요청 작성]
    G --> H[항목별 보완 사항 입력]
    H --> I[보완요청 저장\n상태: S3]
    I --> J[기업에 이메일/SMS 발송]
    J --> K[기업 보완자료 제출 대기]
    K --> L{보완자료 확인}
    L -- 추가 보완 필요 --> G
    L -- 적합 --> F
    F --> M[최종 검토]
    M --> N{최종 결정}
    N -- 승인 --> O[최종 승인 처리\n상태: S4]
    N -- 반려 --> P[반려 처리\n사유 입력]
    O --> Q[승인 이메일/SMS 발송]
    P --> R[반려 이메일/SMS 발송]
    Q & R --> S([심사 완료])
```
